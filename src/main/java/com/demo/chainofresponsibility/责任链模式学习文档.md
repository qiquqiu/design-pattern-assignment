# 责任链模式 - 核心精华

## 一句话理解
**请求沿着链传递，谁想处理就处理，处理完继续传**

## 核心算法
```java
public void handle(int requestLevel, String message) {
    // 1. 判断是否处理
    if (this.level <= requestLevel) {
        write(message);  // 处理
    }

    // 2. 传给下一个（关键）
    if (next != null) {
        next.handle(requestLevel, message);  // 继续传递
    }
}
```

## 核心要点
1. **next指针** - 形成链条的关键
2. **if条件** - 决定是否处理
3. **无论是否处理都要传** - 穿透机制

## 实例记忆
```
领导审批链：
总经理(3) → 部长(2) → 组长(1)

1000元：组长批准 → 部长批准 → 总经理批准
5000元：         部长批准 → 总经理批准
10000元：                  总经理批准
```

## 代码结构
```java
// 1. 抽象处理器
abstract class Handler {
    protected Handler next;  // 下一个处理器
    public abstract void handle();
}

// 2. 具体处理器
class ConcreteHandler extends Handler {
    public void handle() {
        if (能处理) {
            处理请求;
        }
        if (next != null) {
            next.handle();  // 传递给下一个
        }
    }
}

// 3. 客户端
Handler a = new A();
Handler b = new B();
Handler c = new C();
a.next = b;  // 组装链条
b.next = c;
a.handle();  // 发起请求
```

## 本质
**解耦** - 请求发送者不需要知道谁会处理
**灵活** - 可以动态改变处理顺序
**扩展** - 新增处理器不影响原有代码

## 实际场景
- Servlet过滤器链
- 异常处理链
- 审批流程链
- 事件冒泡机制
